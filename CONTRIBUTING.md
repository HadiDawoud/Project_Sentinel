# Contributing to Project Sentinel

Thanks for wanting to help out!

## Getting Started

1. Fork the repo
2. Clone your fork
3. Create a branch for your changes

## Development Setup

```bash
pip install -r requirements.txt
pytest tests/
```

## Making Changes

- Keep changes focused and atomic
- Run tests before submitting
- Write tests for new features

## Commit Messages

Write clear, concise commit messages that explain what you changed and why.

## Pull Requests

- One PR per feature/fix is best
- Include a description of what changed
- Link any related issues

## Questions?

Open an issue and I'll get back to you.
